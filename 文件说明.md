# ComfyUI-VideoHelperSuite 文件说明

## 项目概述
本节点包为 ComfyUI 提供视频处理能力，包含视频合成、字幕、音频加载、批处理等能力。

本次更新重点：
- 路径与平台通用化（Windows / Linux）
- 字幕节点补齐 `frames` 输出
- 新增专业时间线节点（多轨视频/音频/BGM）
- movis 字体跨平台智能识别、样式支持、预览与热刷新
- 保留旧节点并标记为 Deprecated（兼容旧 flow）

## 快速开始
1. 进入 ComfyUI 根目录并安装依赖：
   - `pip install -r custom_nodes/ComfyUI-VideoHelperSuite/requirements.txt`
2. 重启 ComfyUI。
3. 在节点面板中搜索 `VHS_` 前缀节点。

## 项目结构
- `videohelpersuite/nodes.py`：主节点注册与核心节点实现
- `videohelpersuite/caption.py`：字幕引擎、跨平台路径解析、ffmpeg/ffprobe 适配
- `videohelpersuite/video_ops.py`：Movis 视频编排与专业时间线节点
- `videohelpersuite/sf_nodes_pipeline.py`：ShaderFlow 一体化管线节点，包含 `SF Add DepthFlow`
- `videohelpersuite/sf_core.py`：ShaderFlow/DepthFlow 每帧渲染核心
- `videohelpersuite/depthflow_adapter.py`：DepthFlow 路径发现、官方估深器、CUDA/OpenGL 后端、深度图准备与状态映射适配层
- `videohelpersuite/df_nodes_pipeline.py`：DepthFlow 细粒度管线节点
- `videohelpersuite/sharp_engine.py`：独立 Apple SHARP 引擎，自动下载/加载权重、预测 3D Gaussians、生成 SHARP 相机轨迹并渲染视频帧
- `videohelpersuite/sharp_videoops_nodes.py`：独立 SHARP VideoOps 节点，提供一键图像转视频、Gaussian 场景构建、相机预设和模块化视频渲染
- `videohelpersuite/sharp_scene_ops.py`：SHARP/3DGS 场景复用工具，负责 PLY 加载、Gaussian 过滤、变换、合并与保存
- `videohelpersuite/sharp_videoops_extra_nodes.py`：SHARP 扩展节点，提供 PLY 场景处理、相机预设保存/加载、参考视频运镜提取与 DepthFlow 估深复用
- `videohelpersuite/apple_sharp/`：内嵌 Apple SHARP 模型结构与 Gaussian 工具代码，VHS 不要求额外安装 ComfyUI-Sharp 节点包
- `videohelpersuite/utils.py`：ffmpeg 查找、路径校验与工具函数
- `video_formats/`：VideoCombine 的 ffmpeg 编码格式模板

## 核心改动文件说明

### Apple SHARP / VideoOps 独立视频节点
- 新增 `VHS_SHARP_ImageToVideo`：
  - 输入单张 `IMAGE`，自动下载/加载 `apple/Sharp` 权重，直接预测 3D Gaussian 场景并输出视频。
  - 不需要用户手动运行模型准备节点；权重缓存到 ComfyUI `models/sharp`，后续自动复用。
  - 输出 `video_path`、`frames`、`scene`、`ply_path`、`info`。
  - 默认 `render_mode=photo_composite`，在 CUDA + `gsplat` 可用时走 Apple SHARP 官方同类 3D Gaussian rasterization 路径；也可切换 `source_static`、`gaussian_color`、`depth`、`alpha` 做静态原图或调试输出。
  - 默认走无损质量优先：`performance_preset=full`、`splat_quality=quality`、不裁点、不做 alpha 补洞、不叠加原图。
  - `fill_alpha_holes` 默认关闭；用户主动开启时才会用邻近 Gaussian 渲染颜色扩散填补 alpha 小洞，不叠加原图；`source_photo_strength` 默认关闭，用户主动调高时才会在 alpha 几乎为空的像素额外用原图补洞。
  - SHARP 内存策略默认自适应：低系统内存时关闭 encode 特征缓存、建场景后把完整 Gaussian scene 放回 CPU，并释放 SHARP predictor 模型引用，避免 Linux 直接 `Killed`；高配机器可设置 `VHS_SHARP_MEMORY_POLICY=aggressive` 或 `VHS_SHARP_ENCODE_CACHE=1`、`VHS_SHARP_KEEP_SCENE_GPU=1`、`VHS_SHARP_KEEP_MODEL=1` 换取更快重复渲染。该策略不压缩 splat、不降低输出质量，只改变缓存和驻留位置。
  - 新增 `resolution_mode=auto_source/custom/720p/1080p/square_1024/source`，可按源图比例自动决定输出尺寸；`auto_source` 默认限制长边 1024，避免 1280 方图默认过慢。
  - 新增 `performance_preset=full/viewer/draft/balanced/quality/custom`，自动控制 Gaussian 预算；`full` 和 `quality` 保留 SHARP 全部 Gaussians，更接近 GeometryPack 交互 Viewer 的完整 PLY 观感；只有 `custom` 会直接使用手动 `max_gaussians/min_opacity`。
- 新增 `VHS_SHARP_BuildScene`：
  - 将输入图片转换为可复用的 `VHS_SHARP_SCENE` Gaussian 场景。
  - 可控制 `precision`、`focal_length_mm`、`max_gaussians`、`min_opacity`、`save_ply`。
- 新增 `VHS_SHARP_CameraRig`：
  - 提供 SHARP Gaussian 场景相机预设：`official_rotate_forward`、`official_rotate`、`official_swipe`、`official_shake`、`viewer_auto`、`cinematic_orbit`、`portrait_orbit`、`portrait_push`、`window_parallax`、`slow_gallery`、`breathing_closeup`、`dolly_push`、`dolly_pull`、`crane_up`、`truck_left`、`arc_reveal`、`hero_parallax`、`micro_float`、`turntable`、`custom`。
  - 可控制幅度、半径、yaw、pitch、phase，以及 custom 模式下的 yaw/pitch/radius/roll 动画量。
- 新增 `VHS_SHARP_CameraPath`：
  - 支持 JSON 或 CSV `t,x,y,z` 关键帧轨道，输出可直接接入 `VHS_SHARP_RenderVideo` 的 `VHS_SHARP_CAMERA`。
  - 前端 `web/js/sharp_camera_path.js` 提供默认显示的可拖拽 3D 代理轨迹编辑器、录制按钮、深度滑条、锚点选择、清空和镜像操作；编辑区域会随节点尺寸伸缩，并按设备像素比重建 canvas 避免缩放模糊；编辑结果实时写回隐藏的 `path_json` 与 `target_anchor/target_x/target_y/target_z`。
  - 轨迹坐标会按 SHARP 场景深度与焦距自动换算成官方 gsplat 相机位移。
- 新增 `VHS_SHARP_LoadPlyScene`：
  - 读取已有 Apple SHARP/兼容 3DGS PLY，构造成可直接接入 `VHS_SHARP_RenderVideo` 的 `VHS_SHARP_SCENE`。
  - 可复用原始图片作为 `source_image`，也可覆盖 `focal_px` 并按 Gaussian 数量或 opacity 过滤。
- 新增 `VHS_SHARP_SceneProcess`：
  - 对已有 3DGS 场景执行过滤、平移、缩放、Y 轴旋转和可选 PLY 再保存。
- 新增 `VHS_SHARP_MergeScenes`：
  - 合并 2 到 3 个 `VHS_SHARP_SCENE`，可限制合并后的 Gaussian 数量并保存为新 PLY。
- 新增 `VHS_SHARP_MergePlyFolder`：
  - 批量读取目录内匹配 `pattern` 的 PLY 文件，合并成一个 `VHS_SHARP_SCENE` 并可保存为新 PLY。
- 新增 `VHS_SHARP_SaveCameraPreset` / `VHS_SHARP_LoadCameraPreset`：
  - 将任意 `VHS_SHARP_CAMERA` 保存到 `output/sharp_videoops/presets/*.json`，后续工作流可按名称或路径加载复用。
- 新增 `VHS_SHARP_ReferenceVideoToCamera`：
  - 输入参考视频帧 `IMAGE` 批次，使用 OpenCV 光流/仿射估计提取平移与缩放趋势，生成可直接应用到 SHARP 渲染的关键帧相机轨道。
  - 深度图复用 `depthflow_adapter.estimate_depth()`，可调用本地 DepthFlow 的 Depth Anything V1/V2/V3、DepthPro、ZoeDepth；也可输入外部 `reference_depth`。
  - 输出 `camera`、`path_json`、首帧参考 `depth` 和 `info`，用于把参考视频的运动趋势套到 SHARP 3DGS 场景。
- 新增 `VHS_SHARP_RenderVideo`：
  - 输入 `VHS_SHARP_SCENE + VHS_SHARP_CAMERA`，输出视频和帧序列。
  - 支持分辨率、fps、duration、splat_size、opacity_gain、exposure、gamma、背景色和编码器控制。
  - 新增 `render_backend=auto/gpu/cpu` 与 `splat_quality=point/fast/balanced/quality`；CUDA 环境优先使用官方 `gsplat` 光栅化，`balanced/quality` 使用 antialiased 光栅化，非 CUDA 或缺依赖时不再输出扭曲照片视差 fallback，而是报明确错误或由用户手动选择 `source_static`。
  - 新增 `render_mode`、`fill_alpha_holes` 与 `source_photo_strength`，可在官方彩色渲染、纯 Gaussian 彩色、深度可视化、alpha 覆盖图、可选邻域补洞和可选原图补洞之间切换。
- SHARP VideoOps 独立于现有 DepthFlow/ShaderFlow/Movis；注册失败只跳过 SHARP 节点，不影响任何现有节点。
- 模块化节点输出的 `video_path` 和 `frames` 可接入现有 VideoOps/Movis 流程；一键节点则直接输出最终视频。
- `requirements.txt` 新增 `huggingface_hub`、`plyfile` 与 Linux CUDA 环境可用的 `gsplat==1.5.3`，分别用于自动下载 Apple SHARP 权重、可选 PLY 导出和官方风格视频渲染。
- Console 日志增强：
  - 输出模型路径、dtype、加载缓存命中、源图尺寸、SHARP 内部尺寸、焦距换算、encode/decode/unproject 耗时、Gaussian 过滤数量、官方 `gsplat` 是否启用、渲染设备、输出尺寸、megapixels、逐段帧进度、ffmpeg 编码耗时与文件大小。
  - 生成时不再只依赖节点进度条，可在终端判断当前处于下载、模型加载、推理、渲染还是编码阶段。
- Movis/字幕内存策略：
  - `MovisAutoCaptionTimeline` 默认不常驻 stable-whisper 模型；每次识别后会把 segment 转为轻量 dict、清理识别结果对象、释放 CUDA 缓存，避免前序 SHARP/渲染节点后的残留峰值导致系统 `Killed`。
  - 高配环境可设置 `VHS_MOVIS_MEMORY_POLICY=aggressive` 或 `VHS_MOVIS_KEEP_ASR_MODEL=1` 让 ASR 模型常驻以提升多段重复识别速度；默认 `auto` 更偏向稳定。
  - `VideoCaptions`、`VideoGentleCaptions`、`VideoGentleCaptionsPro` 在字幕烧录结束后释放 ASR 模型和 CUDA 缓存。
- 跨平台策略：
  - 路径全部通过 `folder_paths`、`pathlib` 与 `ffmpeg_path` 解析，避免硬编码平台路径。
  - SHARP 推理由 ComfyUI 的 PyTorch 与 `comfy.model_management` 决定；官方质量视频渲染依赖 CUDA + `gsplat`，MPS/CPU 不会伪造官方效果，需要手动选择静态或调试输出模式。
  - 若缺少 torch、huggingface_hub、plyfile、ffmpeg 或本机算子不支持，会在节点执行时报明确异常，不影响其他 VHS 节点注册。

### DepthFlow / ShaderFlow 管线
- `DepthFlow Generator` 默认 `render_backend=opengl`；`auto/cuda` 仍保留原语义，已保存工作流中的显式 backend 不会被强制改写。
- `DepthFlow Generator` 的 OpenGL 子进程命令会探测当前 DepthFlow CLI 支持的动画与参数，避免继续向旧 CLI 发送不存在的 `--intensity` 或不支持的运镜命令。
- `DepthFlow Generator` 的 OpenGL `main` 命令不再追加错误的 `-r` 与 `--format` 参数；视频格式由 `-o` 输出文件后缀决定，避免与 ShaderFlow 的 `ratio -r` 参数冲突。
- `DepthFlow Generator` 的 OpenGL 路径遇到 CLI 未暴露的估深器时，会在 VHS 进程内通过 `depthflow_adapter.estimate_depth()` 生成临时深度图并用 `input -d` 传给原生 OpenGL 渲染，不再静默改用 `da2`。
- `DepthFlow Generator` 会在调用 `.venv_depthflow` 子进程前修补已安装旧 runtime 的 `DepthScene._load_inputs()` resize 兼容问题，避免用户必须手动重建虚拟环境。
- 本地 DepthFlow fork 新增 `depthflow/vhs_animation.py`，通过 `scene.py` 的小型导入钩子注册 VHS 扩展运镜，补齐 `zoom/dolly/orbital` 与 `intensity/reverse/smooth/loop/phase/steady/isometric` 参数，同时降低后续上游 merge 冲突。
- `SF Add DepthFlow` 的 OpenGL 分支会优先使用 `depthflow.vhs_animation`，与 DepthFlow Generator 共享扩展运镜；缺少该扩展时仍回退到上游 `depthflow.animation`。
- `SF Add DepthFlow` 现在通过 `depthflow_adapter.py` 统一调用本地 DepthFlow 源码与估深器，不再在节点内部复制 HuggingFace fallback 逻辑。
- CUDA 作为默认优先后端；CUDA 不可用时尝试 OpenGL/headless，二者都不可用时会报清晰错误。只有显式选择 `backend_policy=allow_cpu` 才允许低质量 CPU fallback。
- DepthFlow 默认质量改为还原优先：`quality_pct=100`、`ssaa=2`，可手动升到 `ssaa=4`；默认不启用非官方边缘模糊。
- `SF Add DepthFlow` 新增 `quality_pct`、`depth_normalize`、`input_depth_invert`、`edge_mode`、`da3`、`depth_model_size`、`da3_resolution`、`depth_postprocess`、`allow_luminance_depth` 等控制项。
- 外部深度图默认只做安全自动缩放，不再强制 min/max 重归一化；需要强制归一化时使用 `depth_normalize=minmax`。
- `SF Add DepthFlow` 暴露 DepthFlow 细粒度状态参数：`height_mode`、`depth_height`、`focus_depth`、`zoom`、`dolly`、`offset_x/y`、`center_x/y`、`origin_x/y`、`mirror`。
- `SF Add DepthFlow` 暴露 DepthFlow 后期参数：vignette、lens、blur、color 全套主要控制项，并继续保留 CUDA/OpenGL 后端策略、SSAA、质量档、边缘软化与深度平滑。
- `sf_core.py` 对 CUDA 与 OpenGL 统一应用这些状态/后期参数；`offset_x/y` 作为动画输出后的额外偏移叠加，不会覆盖原生运动。
- `DF SetState` 补齐 mirror/center/origin，`DF SetPostFX` 补齐 lens quality/decay、blur exponent/quality/directions、color gamma/grayscale。
- `DF Render` 的 CUDA 路径现在完整应用 DepthState 与 PostFX 参数，避免节点面板参数存在但实际渲染未生效。
- 本地 DepthFlow CUDA 渲染器的 PyTorch fallback backward refinement 已按 GLSL `DepthMake()` 修正，保存第一步退到表面外的候选位置，降低 native kernel 不可用时的物体边缘拉伸。
- 本地 DepthFlow CUDA 渲染器的 Vertical/Horizontal/Circle 动画幅度和相位已对齐 `depthflow.animation`；`sf_core.py` 的 OpenGL 桥接也修正 Circle 的 x/y 相位，避免 CUDA/OpenGL 轨迹不一致。
- `shaderflow_bridge.DynamicNumber` 新增标量提取接口，`SF Add Spectrum`、`SF Smooth`、`SF Pipeline Render` 中的平滑逻辑改用 `next_scalar()`，避免新版 numpy 下 `float(array)` 抛出标量转换错误。

### 2) `videohelpersuite/video_ops.py`
- 新增可管线化节点组（非 JSON 文本拼装）：
  - `VHS_MOVIS_CreateTimeline`
  - `VHS_MOVIS_AddVideoTrack`
  - `VHS_MOVIS_AddImageTrack`
  - `VHS_MOVIS_MergeTimeline`（并联分支合流）
  - `VHS_MOVIS_AddImageTrackSequence`
  - `VHS_MOVIS_AddImageMotionTrack`（单图分段运动）
  - `VHS_MOVIS_AddVideoMotionTrack`（视频关键帧运动）
  - `VHS_MOVIS_AddAudioTrack`
  - `VHS_MOVIS_SetBGM`
  - `VHS_MOVIS_AddTextOverlay`
  - `VHS_MOVIS_RenderTimeline`
  - `VHS_MOVIS_Assemble`（一体化快速装配）
  - `VHS_MOVIS_UniversalStudio`（全场景专业装配）
- 多节点可串联传递 `timeline`，最终渲染输出：
  - `video_path`（STRING）
  - `audio`（AUDIO）
  - `frames`（INT）
  - `duration`（FLOAT）
- 输入兼容增强：
  - 视频输入同时支持 `video_path`（STRING）与 `VHS_FILENAMES`（来自 VHS_VideoCombine）
  - 音频输入同时支持 `audio_path/bgm_path`（STRING）与原生 `AUDIO`（ComfyUI）
  - 当路径字符串留空时，会自动回退到连接的原生输入（若存在）
- 时间线能力增强：
  - 支持“同一时间并联多轨”：多个元素设置相同 `start` 即可并行
  - 支持 `VHS_MOVIS_MergeTimeline` 将两条 timeline 分支合流，可为第二条分支设置偏移
  - `VHS_MOVIS_AddImageTrackSequence` 新增 `durations_csv`，支持每张 tensor 图像独立时长
  - `VHS_MOVIS_AddImageTrack` 仍可用于单张图片长时间停留显示
  - `VHS_MOVIS_AddImageMotionTrack` 支持单张图片按关键帧时间（movis Motion）做位置/缩放/旋转/透明度连续动画
  - `VHS_MOVIS_AddVideoMotionTrack` 支持视频片段按关键帧时间做位置/缩放/旋转/透明度连续动画
  - `VHS_MOVIS_SmartMerge` 支持智能合并 timeline（overlay/concat/smart）并自动处理 BGM 规则
  - `VHS_MOVIS_AddImageTrack` / `VHS_MOVIS_AddImageMotionTrack` 现支持路径与原生 `IMAGE` 双输入
  - 图片新增画布适配策略：`transform_only` / `contain` / `cover` / `center_inside` / `stretch`
    - `contain`：完整保留图片，按比例缩放后留边
    - `cover`：填满画布，按锚点裁剪
    - `center_inside`：不放大，仅在超画布时缩小
    - `stretch`：拉伸填满画布（可能变形）
  - 新增锚点控制：`image_anchor_x/image_anchor_y`（0~1）与填充色 `image_bg_color`
  - 修复 Movis 渲染黑屏问题：`position_x/position_y` 仍保持节点侧归一化输入语义，但在渲染阶段自动换算为 movis 所需像素坐标
  - 修复关键帧运动坐标语义：`VHS_MOVIS_AddImageMotionTrack` / `VHS_MOVIS_AddVideoMotionTrack` 的位置关键帧同样自动按画布尺寸换算
  - 修复链式串联只输出单段时长的问题：`VHS_MOVIS_AddVideoTrack` 新增 `placement_mode`
    - `append`：默认值，自动接到当前 timeline 末尾，适合节点串联
    - `absolute`：严格使用 `start` 作为绝对时间，适合多轨叠加/手动编排
  - 修复顶部进度条“秒满”问题：渲染阶段已接入逐帧进度更新，顶部进度会随实际视频写出推进，而非仅反映时间线组装阶段
- 恢复 movis 控制台渲染日志：保留 movis 原生 `tqdm` 输出，并通过 writer 代理同步 ComfyUI 进度条（两者并存）
  - 修复前端数值控件异常：`web/js/VHS.core.js` 中自定义 `VHSFLOAT` / `VHSINT` 控件的边界判断与步进逻辑已修正，避免点击或输入后参数变成 `NaN`
- 增强参数健壮性：关键渲染参数统一做安全转换与范围夹取（防 NaN/Inf/越界），降低异常输入导致渲染失败概率
- 为 movis 与 caption/gentle 关键节点补齐 tooltip 与路径预校验，提升可用性与可诊断性
