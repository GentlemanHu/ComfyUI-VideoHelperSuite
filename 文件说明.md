# ComfyUI-VideoHelperSuite 文件说明

## 项目概述
本节点包为 ComfyUI 提供视频处理能力，包含视频合成、字幕、音频加载、批处理等能力。

本次更新重点：
- 路径与平台通用化（Windows / Linux）
- 字幕节点补齐 `frames` 输出
- 新增专业时间线节点（多轨视频/音频/BGM）
- 保留旧节点并标记为 Deprecated（兼容旧 flow）

## 快速开始
1. 进入 ComfyUI 根目录并安装依赖：
   - `pip install -r custom_nodes/ComfyUI-VideoHelperSuite/requirements.txt`
2. 重启 ComfyUI。
3. 在节点面板中搜索 `VHS_` 前缀节点。

## 项目结构
- `videohelpersuite/nodes.py`：主节点注册与核心节点实现
- `videohelpersuite/caption.py`：字幕引擎、跨平台路径解析、ffmpeg/ffprobe 适配
- `videohelpersuite/video_ops.py`：Movis 视频编排与专业时间线节点
- `videohelpersuite/utils.py`：ffmpeg 查找、路径校验与工具函数
- `video_formats/`：VideoCombine 的 ffmpeg 编码格式模板

## 核心改动文件说明

### 1) `videohelpersuite/caption.py`
- 移除硬编码 Windows ffmpeg 路径。
- 通过 `utils.ffmpeg_path` + `PATH` 自动解析 ffmpeg/ffprobe。
- 新增 `resolve_media_path`，支持相对/绝对/带引号路径，跨平台解析。
- `GentleCaption` 新增：
  - `make_video(...)`：统一字幕渲染入口
  - `parse_caption_params(...)`：安全解析字幕参数（JSON / Python 字典）
  - `estimate_frames(...)`：估算帧数
- 保留兼容接口 `makeVideo(...)` 供旧节点调用。

### 2) `videohelpersuite/video_ops.py`
- 新增可管线化节点组（非 JSON 文本拼装）：
  - `VHS_MOVIS_CreateTimeline`
  - `VHS_MOVIS_AddVideoTrack`
  - `VHS_MOVIS_AddImageTrack`
  - `VHS_MOVIS_MergeTimeline`（并联分支合流）
  - `VHS_MOVIS_AddImageTrackSequence`
  - `VHS_MOVIS_AddImageMotionTrack`（单图分段运动）
  - `VHS_MOVIS_AddVideoMotionTrack`（视频关键帧运动）
  - `VHS_MOVIS_AddAudioTrack`
  - `VHS_MOVIS_SetBGM`
  - `VHS_MOVIS_AddTextOverlay`
  - `VHS_MOVIS_RenderTimeline`
  - `VHS_MOVIS_Assemble`（一体化快速装配）
  - `VHS_MOVIS_UniversalStudio`（全场景专业装配）
- 多节点可串联传递 `timeline`，最终渲染输出：
  - `video_path`（STRING）
  - `audio`（AUDIO）
  - `frames`（INT）
  - `duration`（FLOAT）
- 输入兼容增强：
  - 视频输入同时支持 `video_path`（STRING）与 `VHS_FILENAMES`（来自 VHS_VideoCombine）
  - 音频输入同时支持 `audio_path/bgm_path`（STRING）与原生 `AUDIO`（ComfyUI）
  - 当路径字符串留空时，会自动回退到连接的原生输入（若存在）
- 时间线能力增强：
  - 支持“同一时间并联多轨”：多个元素设置相同 `start` 即可并行
  - 支持 `VHS_MOVIS_MergeTimeline` 将两条 timeline 分支合流，可为第二条分支设置偏移
  - `VHS_MOVIS_AddImageTrackSequence` 新增 `durations_csv`，支持每张 tensor 图像独立时长
  - `VHS_MOVIS_AddImageTrack` 仍可用于单张图片长时间停留显示
  - `VHS_MOVIS_AddImageMotionTrack` 支持单张图片按关键帧时间（movis Motion）做位置/缩放/旋转/透明度连续动画
  - `VHS_MOVIS_AddVideoMotionTrack` 支持视频片段按关键帧时间做位置/缩放/旋转/透明度连续动画
  - `VHS_MOVIS_SmartMerge` 支持智能合并 timeline（overlay/concat/smart）并自动处理 BGM 规则
  - `VHS_MOVIS_AddImageTrack` / `VHS_MOVIS_AddImageMotionTrack` 现支持路径与原生 `IMAGE` 双输入
  - 图片新增画布适配策略：`transform_only` / `contain` / `cover` / `center_inside` / `stretch`
    - `contain`：完整保留图片，按比例缩放后留边
    - `cover`：填满画布，按锚点裁剪
    - `center_inside`：不放大，仅在超画布时缩小
    - `stretch`：拉伸填满画布（可能变形）
  - 新增锚点控制：`image_anchor_x/image_anchor_y`（0~1）与填充色 `image_bg_color`
  - 修复 Movis 渲染黑屏问题：`position_x/position_y` 仍保持节点侧归一化输入语义，但在渲染阶段自动换算为 movis 所需像素坐标
  - 修复关键帧运动坐标语义：`VHS_MOVIS_AddImageMotionTrack` / `VHS_MOVIS_AddVideoMotionTrack` 的位置关键帧同样自动按画布尺寸换算
  - 修复链式串联只输出单段时长的问题：`VHS_MOVIS_AddVideoTrack` 新增 `placement_mode`
    - `append`：默认值，自动接到当前 timeline 末尾，适合节点串联
    - `absolute`：严格使用 `start` 作为绝对时间，适合多轨叠加/手动编排
  - 修复顶部进度条“秒满”问题：渲染阶段已接入逐帧进度更新，顶部进度会随实际视频写出推进，而非仅反映时间线组装阶段
- 恢复 movis 控制台渲染日志：保留 movis 原生 `tqdm` 输出，并通过 writer 代理同步 ComfyUI 进度条（两者并存）
  - 修复前端数值控件异常：`web/js/VHS.core.js` 中自定义 `VHSFLOAT` / `VHSINT` 控件的边界判断与步进逻辑已修正，避免点击或输入后参数变成 `NaN`
- 增强参数健壮性：关键渲染参数统一做安全转换与范围夹取（防 NaN/Inf/越界），降低异常输入导致渲染失败概率
- 为 movis 与 caption/gentle 关键节点补齐 tooltip 与路径预校验，提升可用性与可诊断性
- 修复 `VHS_MOVIS_SetBGM.trim_to_video_length` 被非视频轨道时长“拉长”的问题：
  - 渲染时新增 `_timeline_video_duration()`，仅统计 `video_tracks` 总时长。
  - 当 `trim_to_video_length=true` 时，渲染 `duration` 现在优先严格使用视频总时长，避免 BGM/字幕/外部音轨把总时长抬高导致尾部黑屏。
  - 当无视频轨道时，自动回退到原内容时长逻辑，保持兼容。
- `MovisTimelinePro` 仍保留，作为兼容入口。
- 新增 `VHS_MOVIS_UniversalStudio`：
  - 一节点处理多视频 + 多音频 + 可选 BGM
  - 支持 `concat / overlay / custom_start` 三种编排模式
  - 支持视频/音频的 starts/durations/source_starts/level 批量 CSV 控制
  - 视频输入支持 `videos` 字符串列表或 `VHS_FILENAMES` 自动回退
  - 外部音频与 BGM 支持路径或 ComfyUI `AUDIO` 输入自动回退
  - 自动 Ducking：存在前景音轨时自动压低 BGM
  - 输出包含 `video_path/audio/frames/duration/timeline`，可继续二次管线处理
- 旧节点 `CompositeMedia` / `CompositeMultiVideo` 保留，内部适配到新时间线流程并标注 Deprecated。

### 3) `videohelpersuite/nodes.py`
- 调整自定义能力导入策略：按模块分组导入，避免单个依赖失败导致全部节点失效。
- `VideoCaptions` 与 `VideoGentleCaptions` 统一使用 `GentleCaption` 路径/渲染链路。
- 字幕节点输出新增 `frames`。
- 修复 `VHS_MOVIS_AutoCaptionTimeline` 智能音频源选择策略：
  - 显式 `audio_path` 最高优先级（保持原有行为）。
  - 未显式传入时，优先使用 `timeline.audio_tracks`。
  - `audio_tracks` 为空或不可用时，自动回退到 `timeline.bgm`。
  - 仅在需要时才尝试使用 `video_tracks` 的源音轨，并先做音频流探测。
  - 对无音频流视频（如纯 DepthFlow mp4）不再强行转写，避免 ffmpeg `Output file does not contain any stream`。
  - 兼容旧工作流：若视频本身有音轨且启用 `use_source_audio`，仍可按片段窗口智能对齐生成字幕。
- 修复 `AutoCaptionTimeline` 在 `SetBGM.source_start>0` 时“识别有字幕但画面不显示”的问题：
  - BGM 分支映射字幕时改为使用 `timeline.bgm.source_start` 作为源时间零点，避免整体字幕被错误平移到视频可见窗口外。
  - 新增可见窗口计算（优先 `video_tracks` 总窗口，其次 `audio_tracks`），BGM 字幕只写入可见时间段，避免无效字幕写入远端时间轴。
  - 典型场景：多个无音频视频片段 + 单条 BGM（含 source_start）可稳定生成并显示字幕。
- 性能优化（长 BGM + trim 场景）：
  - 当 `timeline.bgm.trim_to_video_length=true` 且走 BGM 识别分支时，`AutoCaptionTimeline` 会先按可见窗口裁出音频片段再识别（而非全曲识别）。
  - 这样可显著减少超长 BGM 的无效转写耗时。
  - 若窗口裁切失败会自动回退全量识别，保证兼容性与稳定性。
- 新增 `VHS_MOVIS_AutoCaptionTimeline.auto_layout`（默认开启）：
  - 根据 `CreateTimeline` 的 `canvas.width/height` 与长宽比自动适配字幕字号与底部安全区（宽屏/窄屏/超宽均可用）。
  - 默认 `position_y=0.9` 时会自动落到安全区，降低字幕越界或压边概率。
  - 支持完全自定义：关闭 `auto_layout` 后，严格使用你手动设置的 `position_x/position_y` 与样式字号。
- 新增 `custom_override_json` 高优先级覆盖策略（默认空，不影响旧流程）：
  - 布局策略：`layout_mode=auto_safe|auto_bottom|manual`
  - 布局微调：`safe_margin_ratio`、`font_scale`、`position_x`、`position_y`、`font_size/Fontsize`
  - 音频来源策略：`audio_source_strategy=auto|mixed_priority|explicit_only|audio_tracks_only|bgm_only|video_only`
  - 强制路径覆盖：`force_audio_path`、`force_video_path`
  - 所有覆盖项仅在 `custom_override_json` 显式提供时生效，优先级高于 preset 与普通参数。
- `VHS_MOVIS_AutoCaptionTimeline` 现已在节点面板直接暴露策略参数（不必只靠 JSON）：
  - `layout_mode`: `auto_safe | auto_bottom | manual`
  - `audio_source_strategy`: `auto | mixed_priority | explicit_only | audio_tracks_only | bgm_only | video_only`
  - `safe_margin_ratio`: 字幕安全边距（0 表示自动）
  - `font_scale`: 字号缩放
  - 仍保留 `custom_override_json` 最高优先级覆盖（默认空字符串不生效）。
  - 为避免已有 workflow 大面积重连，以上新增策略参数已放在 `optional` 并在函数签名提供默认值，旧流可直接继续跑。
- 新增节点映射：
  - `VHS_MOVIS_CreateTimeline`
  - `VHS_MOVIS_AddVideoTrack`
  - `VHS_MOVIS_AddImageTrack`
  - `VHS_MOVIS_MergeTimeline`
  - `VHS_MOVIS_AddImageTrackSequence`
  - `VHS_MOVIS_AddImageMotionTrack`
  - `VHS_MOVIS_AddVideoMotionTrack`
  - `VHS_MOVIS_AddAudioTrack`
  - `VHS_MOVIS_SetBGM`
  - `VHS_MOVIS_AddTextOverlay`
  - `VHS_MOVIS_RenderTimeline`
  - `VHS_MOVIS_Assemble`
  - `VHS_MOVIS_UniversalStudio`
  - `VHS_MOVIS_TimelinePro`
- 旧映射保留：
  - `VHS_MOVIS_COMPOSITE`（Deprecated）
  - `VHS_MOVIS_MultiVideo`（Deprecated）

## 技术栈
- Python
- ffmpeg / ffprobe
- movis
- stable-whisper / stable-ts
- ComfyUI 节点系统

## 环境要求
- 建议 Python 3.10+
- 系统可用 ffmpeg / ffprobe（或通过 `VHS_FORCE_FFMPEG_PATH` 指定）
- 若使用字幕功能，需安装 stable-whisper 依赖

## 注意事项
- AI 生成代码已做语法级检查，但请在你的实际工作流中做完整验证（不同编码器、媒体容器、长视频、无音轨视频等场景）。
- Deprecated 节点保留用于兼容旧 flow，建议新工作流优先使用 `VHS_MOVIS_TimelinePro`。
